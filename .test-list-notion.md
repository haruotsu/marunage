# PR-201 Notion source plugin — t_wada test list

> Drive Red → Green → Refactor in this order. Tick each entry when the matching
> Go test goes green. New cases discovered mid-implementation get appended at the
> bottom of the relevant section.

## A. Client interface + fake (internal/source/notion/client.go, fake.go)

- [ ] C1. fakeClient returns configured pages from `QueryDatabase` (mirrors the
        real Notion `POST /v1/databases/{id}/query` payload).
- [ ] C2. fakeClient errors when configured (lets List tests assert error
        propagation without standing up an httptest server).
- [ ] C3. fakeClient returns 401-equivalent typed error so AuthStatus can map it
        to AuthRevoked.

## B. Plugin.List (internal/source/notion/notion.go)

- [ ] L1. List returns one source.Task per page; ExternalID = page.id (UUID),
        Source = "notion", Title from the page's title property.
- [ ] L2. List on an empty database returns an empty slice and no error.
- [ ] L3. List propagates a Client error verbatim.
- [ ] L4. List sets Done = page.archived (true → done, false → open).
- [ ] L5. List populates SourcePath from page.url so `marunage show` can deep-
        link to Notion.
- [ ] L6. List populates RawMetadata with last_edited_time and database_id so
        downstream materialisation can reconstruct provenance.
- [ ] L7. List returns ErrDatabaseIDRequired when constructed without
        WithDatabaseID — consistent with markdown's ErrNoFilesConfigured.

## C. Plugin.Since (last_edited_time checkpoint)

- [ ] S1. First Since (no stored checkpoint) returns every page and persists
        checkpoint = max(last_edited_time) into the Checkpointer.
- [ ] S2. Subsequent Since returns only pages whose last_edited_time is strictly
        greater than the stored checkpoint.
- [ ] S3. Since without WithCheckpointer falls back to List behaviour (no
        checkpoint to compare against — degrade to full scan, same contract as
        markdown).
- [ ] S4. Since on an empty result set leaves the existing checkpoint
        untouched (do not regress to zero).
- [ ] S5. Since updates the checkpoint only when the new max is later than the
        stored value (idempotent / monotone).
- [ ] S6. Since uses the Notion API filter
        `{"timestamp":"last_edited_time","last_edited_time":{"on_or_after":...}}`
        so we never page through the whole database again.

## D. Plugin.Setup / AuthStatus

- [ ] D1. AuthStatus returns AuthNotConfigured when the secrets store has no
        token under the documented key.
- [ ] D2. AuthStatus returns AuthAuthenticated when a token is present AND a
        smoke probe (Client.WhoAmI / Users.me) succeeds.
- [ ] D3. AuthStatus returns AuthRevoked when the smoke probe surfaces the
        Notion 401-equivalent typed error.
- [ ] D4. AuthStatus returns AuthExpired on the typed "token expired" Client
        error (OAuth flows surface these separately from revoked).
- [ ] D5. Setup with NonInteractive=true and a TokenProvider that returns a
        non-empty string writes the token to secrets and returns nil.
- [ ] D6. Setup with NonInteractive=true and a TokenProvider returning empty
        string returns ErrTokenRequired.
- [ ] D7. Setup without a SecretsStore configured returns ErrNoSecretsConfigured.

## E. Plugin.Adder / Completer / Deleter (bidirectional, optional)

- [ ] E1. Add creates a page in the configured database (Client.CreatePage gets
        called once with the supplied title) and returns a source.Task whose
        ExternalID matches the new page id.
- [ ] E2. Complete archives the page (Client.UpdatePage with archived=true).
- [ ] E3. Delete archives the page (same call as Complete — Notion has no
        permanent delete; the plugin treats archive as the user-visible delete
        and documents the behaviour). UPDATE: distinguish via different audit
        wording? Keep it simple: Complete sets a checkbox if WithStatusProperty;
        Delete archives.
