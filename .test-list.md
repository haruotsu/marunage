# PR-102 reflection hook — t_wada test list

> Drive Red → Green → Refactor in this order. Tick each entry when the
> matching Go test goes green. New cases discovered mid-implementation get
> appended at the bottom of the relevant section.

## A. internal/store SetReflection

- [x] R1. `SetReflection` writes reflection text to an existing row; other columns untouched.
- [x] R2. `SetReflection` on a missing id returns `ErrNotFound`.
- [x] R3. `SetReflection` accepts an empty string (clears the column).
- [x] R4 (post-design-review): `SetReflection` rejects non-terminal states with `ErrInvalidTransition` so a `marunage reopen` race cannot pin a stale reflection onto a re-pending row.

## B. internal/dispatch reflection hook (reflection.go)

- [x] N1. `New` fails `ErrInvalidConfig` when `WithStore` is omitted.
- [x] N2. `New` fails `ErrInvalidConfig` when `WithCmux` is omitted.
- [x] N3. `New` fails `ErrInvalidConfig` when `WithSkill` is empty.
- [x] N4. `New` fails `ErrInvalidConfig` when `WithWorkspaceDirs` is omitted.
- [x] N5. `New` rejects `WithSampleRate` outside [0,1].
- [x] N6. `New` defaults timeout to 5m, clock to time.Now, sample_rate to 1.0, auditor to NopAuditor.

## C. Sampling

- [x] S1. `WithSampleRate(0)` => OnDone never sends.
- [x] S2. `WithSampleRate(1)` => OnDone always sends. (covered by H3 via default rate=1.0 + H7 fan-out.)
- [x] S3. `WithSampler` overrides the rate-based sampler. (last-writer-wins.)

## D. Async hook behaviour (Reflector.OnDone + Wait)

- [x] H1. `OnDone` with empty `task.WS` is a no-op (no Send, no SetReflection, no goroutine started).
- [x] H2. `OnDone` with sampler.False is a no-op.
- [x] H3. `OnDone` calls `cmux.Send(ws, prompt)` with prompt containing the skill body and the reflection-sentinel write instruction.
- [x] H4. After `Send`, the goroutine polls for `<workspaceDir>/.reflection` and persists the trimmed contents via `store.SetReflection`.
- [x] H5. Timeout: when `.reflection` never appears, the goroutine exits before the parent ctx fires; no SetReflection call; audit `reflection.timeout` recorded.
- [x] H6. Parent ctx cancel propagates to the goroutine; it exits without calling SetReflection; audit `reflection.cancel` recorded.
- [x] H7. `Wait` blocks until every dispatched goroutine exits.
- [x] H8. `cmux.Send` failure: audit `reflection.fail`; no SetReflection call.
- [x] H9. Audit `reflection.start` recorded with `Key=task:<id>`, `Value=<ws>` immediately before the Send.
- [x] H10 (post-design-review): orphan `.reflection.tmp` (mid-publish) does not trigger SetReflection — the polled file is the post-rename `.reflection`, so half-written reads cannot land in `tasks.reflection`.
- [x] H11 (post-design-review): the prompt body uses a heredoc rather than `printf '%s'` so a reflection containing `%`, `"`, `\`, or newlines round-trips verbatim. (covered by H3 string-shape assertion.)
- [x] H12 (review-fix-loop iter1): `cmux.Send` returning `context.Canceled` is classified as `reflection.cancel`, not `reflection.fail`.
- [x] H13 (review-fix-loop iter1): `cmux.Send` returning `context.DeadlineExceeded` is classified as `reflection.timeout`.
- [x] H14 (review-fix-loop iter1): audit Value runs through `logging.Redact` so a `Bearer sk-…` echoed back from cmux stderr does not pin to audit.log.
- [x] H15 (review-fix-loop iter1): the prompt advertises the configured `WithReflectionTimeout`, not the package default.

## E. completion package WithDoneHook integration

- [x] C1. `completion.WithDoneHook(fn)` invokes fn with the task's id immediately after a successful done transition.
- [x] C2. Hook is NOT invoked when MarkDoneWithSummary fails.
- [x] C3. Hook is NOT invoked on the failed (non-zero exit) branch.
- [x] C4. Hook receives the post-done store.Task (so reflection prompt can carry the result_summary if needed). (covered by C1: status/summary/completed_at asserted on the captured task.)
