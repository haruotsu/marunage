# PR-43 atomic sentinel completion detection — t_wada test list

> Drive Red → Green → Refactor in this order. Tick each entry when the matching
> Go test goes green. New cases discovered mid-implementation get appended at the
> bottom of the relevant section.

## A. internal/store additions (atomic done + completed_at)

- [x] S1. `MarkDoneWithSummary` on a row stamps status=done, result_summary=<text>, completed_at=<time>.
- [x] S2. `MarkDoneWithSummary` on a missing id returns `ErrNotFound`.
- [x] S3. `MarkDoneWithSummary` accepts an empty summary (Claude may produce no final text).
- [x] S4. `MarkDoneWithSummary` with zero `time.Time` returns `ErrCompletedAtRequired`.
- [x] S5. `SetCompletedAt` stamps completed_at on a row; missing id returns `ErrNotFound`; zero time returns `ErrCompletedAtRequired`.
- [x] S6. `MarkDoneWithSummary` writes the canonical millisecond ISO8601 layout that `Get` round-trips.

## B. internal/completion package — Watcher.Tick(ctx)

- [ ] W1. No running rows → `Tick` returns nil and never touches the filesystem.
- [ ] W2. Running row without sentinel → `Tick` is a no-op (row stays running, no audit, no store writes).
- [ ] W3. Running row with sentinel `0\n` → row marked done, audit `completion.detect` recorded carrying the task id.
- [ ] W4. Running row with sentinel `1\n` (non-zero exit) → row marked failed, judgment_reason mentions `exit_code=1`, audit `completion.fail`.
- [ ] W5. Running row with sentinel `garbage` (non-numeric) → row marked failed, judgment_reason mentions parse failure, audit `completion.fail`.
- [ ] W6. `.exit_code.tmp` present but `.exit_code` absent → `Tick` is a no-op (atomicity preserved; reader does not consume partial file).
- [ ] W7. `.result_summary` file present alongside `.exit_code` → its trimmed contents become `tasks.result_summary`.
- [ ] W8. `.result_summary` absent on success → result_summary stays empty.
- [ ] W9. Multiple running rows: each one independently checked; one's sentinel does not affect another's outcome.
- [ ] W10. Workspace directory missing entirely → row stays running (PR-44 reaper's job, not ours; no audit fired).
- [ ] W11. Audit `completion.detect` carries `Key="task:<id>"` and `Value="<exit_code>"`.
- [ ] W12. Audit `completion.fail` carries `Key="task:<id>"` and `Value` containing the failure reason.
- [ ] W13. Sentinel write is atomic for the reader: simulating the documented `mv .exit_code.tmp .exit_code` (write tmp then rename) leaves the watcher seeing only the final byte once, never a half-written file.

## C. Watcher construction

- [ ] N1. `New` fails with `ErrInvalidConfig` when `WithStore` is omitted.
- [ ] N2. `New` fails with `ErrInvalidConfig` when `WithWorkspaceDirs` is omitted.
- [ ] N3. `New` defaults clock to `time.Now`, auditor to `config.NopAuditor`, poll interval to 5s.

## D. Watcher.Run(ctx) — long-running loop

- [ ] R1. `Run` invokes `Tick` once per poll interval and continues until ctx is cancelled.
- [ ] R2. ctx cancellation returns nil so the daemon caller can shut down cleanly.
- [ ] R3. `Run` calls `Tick` immediately on entry (does not wait the first poll interval).

## E. Prompt wiring (internal/dispatch/prompt.go)

- [ ] P1. `BuildPrompt` with non-empty `WorkspaceDir` appends a sentinel-write instruction containing the absolute `.exit_code` path.
- [ ] P2. The sentinel instruction includes the documented atomic command pair: `echo $? > .exit_code.tmp` then `mv .exit_code.tmp .exit_code` (so a reader never sees a half-written file).
- [ ] P3. Empty `WorkspaceDir` omits the sentinel instruction (back-compat for callers that have not wired completion yet).
- [ ] P4. The instruction tells Claude to write `.result_summary` (trimmed final summary) before the sentinel rename so Watcher reads it atomically.

## F. Dispatcher wiring (internal/dispatch/dispatch.go)

- [ ] F1. `WithWorkspaceDirs` plumbs the same `Dirs` provider used by Watcher; the dispatcher mkdirs the per-task dir before NewWorkspace and passes the dir into `BuildPrompt`.
- [ ] F2. Mkdir failure leaves the row pending (do not flip to failed; PR-43 wants retry semantics consistent with NewWorkspace failure).
- [ ] F3. Default behaviour with no `WithWorkspaceDirs` keeps PR-42 wire-format intact (no sentinel section, no mkdir).
