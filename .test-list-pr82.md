# PR-82 Slack source plugin — t_wada test list

> Drive Red → Green → Refactor in this order. Tick each entry when the matching
> Go test goes green. New cases discovered mid-implementation get appended at
> the bottom of the relevant section.

## A. Slack Client interface + fake

- [x] A1. `Client` interface declares `FetchMentions(ctx, sinceTS) ([]Message, error)`,
        `FetchDMs(ctx, sinceTS) ([]Message, error)`, `PostDM(ctx, channelID, text) error`,
        `AuthStatus(ctx) (source.AuthStatus, error)`, and `Setup(ctx, NonInteractive bool) error`.
- [x] A2. `fakeClient` records arguments per method and returns canned `Message` slices
        (built only inside the test package).
- [x] A3. `Message` struct exposes `ChannelID`, `ChannelType` ("channel"/"im"), `TS`,
        `ThreadTS`, `UserID`, `Text`, `Permalink` fields.

## B. Plugin construction & name

- [x] B1. `Plugin.Name()` returns `"slack"`.
- [x] B2. `New(WithClient(c), WithCheckpointer(kv), WithIncludeMentions(true), WithIncludeDM(true), WithNotifyChannelID("D123"))`
        round-trips fields onto the struct.
- [x] B3. `New()` with no client uses a `nilClient` whose every method returns
        `ErrClientNotConfigured` so the plugin is still constructible (matches markdown).

## C. List

- [x] C1. `List` with both flags off returns an empty slice without calling Client.
- [x] C2. `List` with only mentions=true calls `FetchMentions` and not `FetchDMs`.
- [x] C3. `List` with only DM=true calls `FetchDMs` and not `FetchMentions`.
- [x] C4. `List` with both true merges results from both calls.
- [x] C5. `List` converts each `Message` into a `source.Task` with
        `Source="slack"`, `ExternalID="<channel>:<ts>"`, `Title=Text` (truncated to 120 chars on first line),
        `Body=Text` when multi-line, and `SourcePath=Permalink`.
- [x] C6. `List` populates `RawMetadata` with `channel_id`, `thread_ts`, `ts`,
        `user_id`, `channel_type` keys. `dm_id` is set only when ChannelType=="im".
- [x] C7. `List` propagates a Client error verbatim (wrapped with the source name).
- [x] C8. `List` calls FetchMentions/FetchDMs with empty sinceTS (List ignores checkpoint).

## D. Since (Sincer)

- [x] D1. `Since(ctx, "")` with no Checkpointer behaves like List with sinceTS="".
- [x] D2. `Since(ctx, "")` with a Checkpointer reads the current checkpoint key
        `slack:last_ts`; on first run (no checkpoint) treats it as empty string.
- [x] D3. `Since(ctx, checkpoint)` forwards the explicit argument to Client.Fetch*,
        ignoring any stored value (the source.Sincer arg wins).
- [x] D4. `Since` after returning at least one message persists the maximum `ts`
        across mentions+DMs into the Checkpointer under `slack:last_ts`.
- [x] D5. `Since` with zero results does NOT touch the checkpoint.
- [x] D6. Maximum-ts comparison is numeric (Slack ts are decimal strings, e.g.
        "1700000000.000100"); larger ts lexicographically only after equal-length compare.
- [x] D7. `Since` propagates Client errors and never advances the checkpoint on error.

## E. Complete (Completer — DM completion notifier)

- [x] E1. `Complete(ctx, "42")` calls `Client.PostDM(ctx, notifyChannelID, "タスク #42 done")`.
- [x] E2. `Complete` with empty externalID returns `ErrInvalidTaskID`.
- [x] E3. `Complete` without `WithNotifyChannelID` set returns `ErrNotifyChannelRequired`.
- [x] E4. `Complete` propagates `Client.PostDM` errors verbatim.

## F. AuthStatus / Setup

- [x] F1. `AuthStatus` forwards to `Client.AuthStatus` and returns its answer.
- [x] F2. `AuthStatus` with no client returns `AuthNotConfigured` (graceful default).
- [x] F3. `Setup(ctx, opts)` forwards `opts.NonInteractive` to `Client.Setup`.
- [x] F4. `Setup` with no client returns `ErrClientNotConfigured`.

## G. Adapter (source.Plugin contract)

- [x] G1. `NewAdapter(p).Name() == "slack"`.
- [x] G2. Adapter satisfies `source.Plugin`, `source.Sincer`, `source.Completer`.
- [x] G3. Adapter does NOT implement `source.Adder` or `source.Deleter`.
- [x] G4. Adapter.List output equals the inner Plugin's slice (lifted into source.Task).
- [x] G5. Adapter.Since forwards its checkpoint argument to the inner Plugin.
- [x] G6. Adapter.Complete forwards externalID untouched.
- [x] G7. Adapter.AuthStatus forwards the result.

## H. Builtin / manifest

- [x] H1. `plugin.toml` declares name=slack, sync_mode=bidirectional, version="0.1.0",
        capabilities = [list, setup, auth-status, since, complete].
- [x] H2. `Manifest()` parses the embedded TOML and surfaces the same data.
- [x] H3. `RegisterBuiltin(r, opts...)` attaches the adapter under "slack" and passes
        `source.ValidateAgainstManifest`.
- [x] H4. Double `RegisterBuiltin` returns `source.ErrPluginAlreadyRegistered`.

## Cross-cutting

- [x] All existing tests (`go test ./...`) stay green.
- [x] `make lint` clean.
