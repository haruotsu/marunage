## A. internal/web Dashboard provider (snapshot aggregation)

- [x] D1. `Snapshot` returns running tasks (id, source, title, ws, started_at, output preview), ordered by started_at ASC, id ASC.
- [x] D2. `Snapshot` truncates the running output preview to the configured byte budget.
- [x] D3. `Snapshot` returns the top-N pending tasks ordered by priority DESC, created_at ASC, id ASC.
- [x] D4. `Snapshot` returns the total pending count distinct from the top-N slice.
- [x] D5. `Snapshot` reports 24h totals: done / failed / skipped within `now - window`.
- [x] D6. `Snapshot` reports per-source 24h breakdown sorted by source name.
- [x] D7. `Snapshot` reports each registered source's auth status and last-listed checkpoint time.
- [x] D8. `Snapshot` records `unknown` auth status when a plugin returns an error.
- [x] D9. `Snapshot` stamps `GeneratedAt` from the injected clock.
- [x] D10. `noopDashboardProvider` returns an empty snapshot with `GeneratedAt` set so handlers never deref nil.
- [x] D11. `Snapshot` propagates store errors with a wrapped panel-specific message.

## B. SQL-backed DashboardStore (production wiring)

- [x] S1. `dashboardSQLStore.Running` selects status='running' rows ordered started_at ASC.
- [x] S2. `dashboardSQLStore.PendingTop` selects pending rows by (priority DESC, created_at ASC, id ASC) and respects limit.
- [x] S3. `dashboardSQLStore.PendingCount` returns the total pending count regardless of limit.
- [x] S4. `dashboardSQLStore.Recent` aggregates done/failed/skipped per source within the window.
- [x] S5. `dashboardSQLStore.SourceCheckpoints` returns the most-recent updated_at per `<source>(_|:)*` key.
- [x] S6. `dashboardSQLStore.Recent` includes rows whose timestamp equals the cutoff (half-open `>=`).
- [x] S7. `dashboardSQLStore.Running` truncates the preview on a UTF-8 boundary so multi-byte runes are not split.
- [x] S8. `dashboardSQLStore.SourceCheckpoints` parses the markdown plugin's `markdown:mtime:<path>` colon namespace.
- [x] S9. `dashboardSQLStore.SourceCheckpoints` keeps the latest updated_at when one source publishes multiple keys.
